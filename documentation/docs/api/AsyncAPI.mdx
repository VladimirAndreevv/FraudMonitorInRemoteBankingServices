# API

## AsyncAPI

Данный API описывает асинхронный обмен событиями о транзакциях между системой дистанционного банковского обслуживания (ДБО) и системой мониторинга подозрительной активности.

Передача выполняется через брокер сообщений Kafka без ожидания ответа от получателя.

```yaml
asyncapi: 3.0.0
info:
  title: API обмена событиями о транзакциях
  version: 1.0.0
  description: Асинхронная передача событий из дистанционного банковского обслуживания в систему мониторинга подозрительной активности

channels:
  transaction:
    address: transactions
    messages:
      TransactionCreated:
        $ref: '#/components/messages/TransactionCreated'
    bindings:
      kafka:
        bindingVersion: '0.5.0'

operations:
  sendTransaction:
    action: send
    channel:
      $ref: '#/channels/transaction'
    summary: Отправка события о транзакции из дистанционного банковского обслуживания в систему мониторинга
    bindings:
      kafka:
        bindingVersion: '0.5.0'
        clientId:
          type: string
          enum:
            - dbr-producer

servers:
  kafkaBroker:
    host: localhost:9092
    protocol: kafka
    description: Kafka broker
    bindings:
      kafka:
        bindingVersion: '0.5.0'

components:
  messages:
    TransactionCreated:
      name: TransactionCreated
      title: Событие о созданной транзакции
      contentType: application/vnd.apache.avro
      payload:
        schemaFormat: 'application/vnd.apache.avro;version=1.9.0'
        $ref: './TransactionCreated.avsc'
```

### Поведение API

- Передача данных выполняется асинхронно
- Отправитель не ждёт немедленного ответа
- Сообщения записываются в Kafka-топик `transactions`
- Система мониторинга получает и обрабатывает событие независимо

### Гарантии доставки

- используется модель `at-least-once`
- возможна повторная доставка сообщения
- обработка на стороне получателя должна быть идемпотентной
- при сбое возможна повторная отправка
- некорректные сообщения могут попадать в DLQ

### Структура события

Сообщение описывает одну транзакцию пользователя и содержит данные, необходимые для анализа риска.

#### Основные поля

- `idEvent` — уникальный идентификатор события
- `TimeEvent` — время создания события
- `SystemSource` — источник события
- `idUser` — идентификатор пользователя
- `idTransaction` — идентификатор транзакции
- `TypeOperation` — тип операции
- `Amount` — сумма транзакции
- `Currency` — валюта операции
- `idDevice` — идентификатор устройства
- `ipAddress` — IP-адрес
- `Geolocation` — геолокация

### Валидация сообщения

Перед обработкой выполняются проверки:

- наличие обязательных полей;
- корректность типов данных;
- уникальность `idEvent`;
- корректность формата времени;
- соответствие Avro-схеме.

При ошибке:

- сообщение помечается как `data_error`;
- сообщение отправляется в DLQ.

### Использование в системе

Событие `TransactionCreated` используется для:

- анализа поведения пользователя;
- применения правил детекции;
- расчёта уровня риска;
- формирования инцидентов;
- построения аналитики и отчётов.
