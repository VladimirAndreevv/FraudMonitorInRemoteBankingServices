# API

## AsyncAPI

Данный API описывает асинхронный обмен событиями о транзакциях между системой дистанционного банковского обслуживания (ДБО) и системой мониторинга подозрительной активности.

Передача выполняется через брокер сообщений Kafka без ожидания ответа от получателя.

```yaml
asyncapi: 3.0.0
info:
  title: API обмена событиями о транзакциях
  version: 1.0.0
  description: Асинхронная передача событий из дистанционного банковского обслуживания в систему мониторинга подозрительной активности

channels:
  transaction:
    address: transactions
    messages:
      TransactionCreated:
        $ref: '#/components/messages/TransactionCreated'
    bindings:
      kafka:
        bindingVersion: '0.5.0'

operations:
  sendTransaction:
    action: send
    channel:
      $ref: '#/channels/transaction'
    summary: Отправка события о транзакции из дистанционного банковского обслуживания в систему мониторинга
    bindings:
      kafka:
        bindingVersion: '0.5.0'
        clientId:
          type: string
          enum:
            - dbr-producer

servers:
  kafkaBroker:
    host: localhost:9092
    protocol: kafka
    description: Kafka broker
    bindings:
      kafka:
        bindingVersion: '0.5.0'

components:
  messages:
    TransactionCreated:
      name: TransactionCreated
      title: Событие о созданной транзакции
      contentType: application/vnd.apache.avro
      payload:
        schemaFormat: 'application/vnd.apache.avro;version=1.9.0'
        $ref: './TransactionCreated.avsc'
```

### Поведение API

1. Передача данных выполняется асинхронно
2. Отправитель не ждёт немедленного ответа
3. Сообщения записываются в Kafka-топик `transactions`
4. Система мониторинга получает и обрабатывает событие независимо

### Гарантии доставки

1. используется модель `at-least-once`
2. возможна повторная доставка сообщения
3. обработка на стороне получателя должна быть идемпотентной
4. при сбое возможна повторная отправка
5. некорректные сообщения могут попадать в DLQ

### Структура события

Сообщение описывает одну транзакцию пользователя и содержит данные, необходимые для анализа риска.

#### Основные поля

1. `idEvent` — уникальный идентификатор события
2. `TimeEvent` — время создания события
3. `SystemSource` — источник события
4. `idUser` — идентификатор пользователя
5. `idTransaction` — идентификатор транзакции
6. `TypeOperation` — тип операции
7. `Amount` — сумма транзакции
8. `Currency` — валюта операции
9. `idDevice` — идентификатор устройства
10. `ipAddress` — IP-адрес
11. `Geolocation` — геолокация

### Валидация сообщения

Перед обработкой выполняются проверки:

1. наличие обязательных полей;
2. корректность типов данных;
3. уникальность `idEvent`;
4. корректность формата времени;
5. соответствие Avro-схеме.

При ошибке:

1. сообщение помечается как `data_error`;
2. сообщение отправляется в DLQ.

### Использование в системе

Событие `TransactionCreated` используется для:

1. анализа поведения пользователя;
2. применения правил детекции;
3. расчёта уровня риска;
4. формирования инцидентов;
5. построения аналитики и отчётов.
