# Технологический стек системы мониторинга подозрительной активности в ДБО

Для реализации системы мониторинга подозрительной активности используется набор технологий, обеспечивающих надёжный приём событий, их хранение, обработку в режиме, близком к реальному времени, а также формирование аналитики и отчётности.

## Основные компоненты

| Компонент | Технология | Назначение |
| --------- | ---------- | ---------- |
| Backend | Python / Java | Реализация бизнес-логики, обработки событий, правил детекции и управления инцидентами |
| Хранилище транзакций | PostgreSQL | Хранение пользовательских, транзакционных и инцидентных данных |
| Аналитическое хранилище | ClickHouse / DWH | Хранение агрегированных данных, отчётности и аналитических выборок |
| Сессии и быстрый доступ | Redis | Быстрый доступ к данным сессий и контексту обработки событий |
| Очередь событий | Kafka | Асинхронная передача событий между компонентами системы |
| API | REST, AsyncAPI | Интеграция с внешними банковскими системами и описание обмена событиями |
| Инфраструктура | Docker | Контейнеризация компонентов системы и упрощение развёртывания |

## Обоснование выбора

1. **Python / Java** используются для реализации серверной логики, так как позволяют строить как прикладные сервисы, так и компоненты для анализа и обработки событий.
2. **PostgreSQL** подходит для транзакционных данных, где важны целостность, согласованность и поддержка связей между сущностями.
3. **ClickHouse / DWH** используются для аналитической нагрузки, так как система должна быстро формировать отчёты и агрегаты по большим объёмам данных.
4. **Redis** применяется для хранения сессионных данных и быстрого доступа к часто используемой информации.
5. **Kafka** обеспечивает асинхронный обмен событиями и устойчивую обработку потока транзакций.
6. **REST** используется для синхронных интеграций, а **AsyncAPI** — для описания сообщений и событийного взаимодействия.
7. **Docker** упрощает развёртывание, переносимость и сопровождение компонентов системы.
